## Phase 3 — SVGテンプレートとレンダリング

**目標**: サンプルデータで PNG を目視確認できる状態にする。

- [ ] `assets/template.svg` を作成（プレースホルダー入りの完成デザイン）

  - ネタバレバッジ領域も含める（`{{SPOILER_BADGE}}`）
- [ ] `src/image.rs` を作成

  - プレースホルダーを `str::replace` で置換
  - `usvg` でパース → `resvg` で `tiny-skia` キャンバスにレンダリング → PNG バイト列を返す
- [ ] `image.rs` のユニットテストを書く（サンプルペイロードを渡してPNGバイト列が返ること）
- [ ] テスト実行時に PNG をファイルに書き出して目視確認

## Phase 4 — ファイル出力・mainの結合

**目標**: CLI として end-to-end で動く。

- [ ] `src/main.rs` を完成させる
  - `std::env::args()` で第1引数（JSON文字列）を取得
  - 引数なし・パース失敗時はエラーJSONを標準出力して exit code 1
  - 環境変数 `OUTPUT_DIR` / `FONT_PATH` / `SVG_TEMPLATE_PATH` を読み込む
  - `image::render()` を呼び出してPNGバイト列を取得
  - `OUTPUT_DIR/{hash}.png` に書き込む
  - 成功時は `{"ok":true,"path":"..."}` を標準出力して exit code 0
- [ ] 仕様書の動作確認コマンドで手動テスト

## Phase 7 — CI/CDパイプライン

**目標**: `main` ブランチへの push で本番サーバーにバイナリが自動デプロイされる。

### 事前準備

- [ ] デプロイ用SSHキーペアを生成し、公開鍵をサーバーの `~/.ssh/authorized_keys` に登録
- [ ] GitHub リポジトリの Settings → Secrets and variables → Actions に以下を登録

| Secret名 | 内容 |
|---------|------|
| `DEPLOY_HOST` | サーバーのホスト名またはIPアドレス |
| `DEPLOY_USER` | SSHログインユーザー（例: `www-data` または `deploy`） |
| `DEPLOY_SSH_KEY` | SSHの秘密鍵（`-----BEGIN OPENSSH PRIVATE KEY-----` ごと） |
| `DEPLOY_PATH` | バイナリの配置先（例: `/usr/local/bin`） |

### ワークフロー作成

- [ ] `.github/workflows/deploy-ogp-generator.yml` を作成

```yaml
name: Deploy ogp-generator

on:
  push:
    branches: [main]
    paths:
      - 'crates/ogp-generator/**'
      - 'Cargo.toml'
      - 'Cargo.lock'

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Install Rust toolchain
        uses: dtolnay/rust-toolchain@stable

      - name: Cache cargo build
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            target
          key: ${{ runner.os }}-cargo-${{ hashFiles('Cargo.lock') }}
          restore-keys: ${{ runner.os }}-cargo-

      - name: Build release binary
        run: cargo build -p ogp-generator --release

      - name: Deploy binary to server
        env:
          SSH_KEY: ${{ secrets.DEPLOY_SSH_KEY }}
        run: |
          install -m 600 /dev/null /tmp/deploy_key
          echo "$SSH_KEY" > /tmp/deploy_key
          scp -i /tmp/deploy_key \
              -o StrictHostKeyChecking=accept-new \
              target/release/ogp-generator \
              ${{ secrets.DEPLOY_USER }}@${{ secrets.DEPLOY_HOST }}:${{ secrets.DEPLOY_PATH }}/ogp-generator
          rm -f /tmp/deploy_key
```

### ポイント

- **`paths:` フィルタ**により、ogp-generator に無関係な変更（他クレートの更新など）ではデプロイが走らない
- exec方式のためサービス再起動不要。バイナリを上書きするだけでよい（次回のキュージョブ実行から新バイナリが使われる）
- キャッシュにより2回目以降のビルドが高速化される
